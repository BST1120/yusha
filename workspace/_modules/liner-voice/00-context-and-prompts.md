# コンテクストとプロンプト — 整理の仕方

AI に調査・執筆を頼むとき、**プロンプト（指示）** と **コンテクスト（材料）** を分ける。

| 種類 | 中身 | 例 |
|------|------|-----|
| **コンテクスト** | 事実・ラフ・制約・参照 | `draft/_log.md`、歌詞、`tracks.json`、参照URL |
| **プロンプト** | やってほしい操作・出力形式 | 「工程だけ」「POPEYE+JJJ」「500字」 |

**ルール:** コンテクストは増やす。プロンプトは短く具体的に。

---

## プロンプト1 — 参照文体の調査（ChatGPT / Cursor 用）

コピーして使う。URL や記事を貼れるときは末尾に追加。

```text
あなたは日本の音楽ライティング調査アシスタントです。

【調査対象】
以下のジャンル・アーティスト・媒体で、ライナーノーツ・制作インタビュー・本人コメントが載っているWeb記事を探し、文体を分析してください。

- 日本語ヒップホップ / インディーラップ（例: JJJ、千人賞、百足、Fuma no KTR、ISSEI、SKRYU など）
- インディー／R&B／バンド系（例: um-hum、ウムスビ — 該当があれば）
- ラブ系・パンク／ポップ系（ユーザー関心: ラブのパンピー、イスギ — 正式表記を確認して検索）
- 雑誌 POPEYE の「音楽のこと。」コラム、Music 特集、Web 記事
- note 等のセルフライナーノーツ（ボカロP、インディー）

【分析してほしい項目】
各ソースごとに:
1. 書き出し（最初の2〜3文）の型
2. 文体: 話し言葉 / 書き言葉 / 距離感
3. 載せている情報の種類（工程、機材、エピソード、意図、感想）
4. ユーモアの入れ方
5. 「説教・論評」になっている部分と「制作メモ」になっている部分の違い
6. その文体を真似するときの DO / DON'T（各3つ）

【出力形式】
表形式 + 各ソースから短い引用（50字以内）を2例ずつ。

【注意】
- 推測で引用を捏造しない。見つからなければ「未確認」と書く。
- 私が目指すのは「制作工程を淡々と、でも柔らかく」書くライナー調。エッセイ調は避けたい。
```

## プロンプト2 — 特定アーティスト深掘り

```text
「{アーティスト名}」の制作インタビューまたはライナーノーツを5本探してください。
サンレコ、Real Sound、SPICE、BARKS、Apple Music 曲コメント、note セルフライナーノーツ等。

各記事について:
- URL
- 誰が書いているか（本人 / ライター）
- 書き出しの全文（最初の段落）
- 制作情報として載っている具体項目（DAW、プラグイン、サンプル元、studio、日数、順番）
- 口調タグ（3語以内: 例「くだけ・具体・ノリ」）

最後に「{アーティスト名}文体モジュール」として、
opening / stance / context-types / vocabulary を各5行以内で提案してください。
```

アーティスト名例（ユーザー指定 — 正式表記要確認）:
- JJJ
- 千人賞
- ラブのパンピー（表記ゆれあり — 検索時に複数パターン試す）
- イスギ
- ウムスビ

## プロンプト3 — POPEYE 系の「柔らかい書き出し」抽出

```text
POPEYE または POPEYE Web の、音楽・カルチャー・ライフスタイル記事から、
「読者を硬く説明しない」書き出しを10個集めてください。

条件:
- 最初の1〜2文だけ抜粋
- なぜ柔らかいか（1行コメント）
- パターン名を付ける（例: 日常から入る / 自白から入る / 場所から入る）

私の制作記録では、説教ではなく「隣で話している」距離感が欲しい。
```

## プロンプト4 — ラフ → 制作記録への変換（Cursor 執筆用）

`draft/_log.md` を貼ったうえで:

```text
添付ラフを、apps/yusha-site/workspace/_modules/liner-voice/ のモジュールに従い、
制作記録（output/latest.txt）に変換してください。

【指定モジュール】
- form: {forms/process-log.md など}
- stance: {stances/jjj-studio.md など}
- opening: {openings/01-cold-open-fact.md など}
- context-types: gear, workflow, anecdote（該当するもの）

【制約】
- 「〜と考えている」「〜ではないだろうか」を原則使わない
- 【当時の意図】のような見出しは使わない（プレーンテキスト、短い段落）
- 800字以内
- 固有名詞（Suno, Cubase, Groove Agent 等）は残す
- 説明より工程と事実

【出力】
latest.txt 用プレーンテキストのみ
```

## プロンプト5 — 書き手臭い診断

```text
以下の制作記録テキストを、01-anti-patterns.md の基準で診断してください。

- 書き手臭い箇所（引用 + 理由）
- 言い換え案（同じ事実を、ライナー調で）
- 削れる段落
- 残すべき固有名詞・工程

テキスト:
---
{ここに本文}
---
```

## 調査結果の取り込み方

1. AI の調査結果で良かった引用 → `references/` にカード追加
2. パターン名が定着 → `openings/` または `stances/` に新モジュール
3. 自分のラフでうまくいった output → `examples/` に保存
