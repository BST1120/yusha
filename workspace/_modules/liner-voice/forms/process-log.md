# Form: process-log（工程ログ）

## 用途
制作記録の**デフォルト**。意図より手順・道具・順番。

## 構造（見出しなし・段落のみ）

1. **書き出し** — 1〜2文（何をしたか）
2. **素材の来路** — どこから音が来たか（Suno、采样、演奏…）
3. **作業** — DAW、プラグイン、操作（箇条書き可）
4. **縛り** — やらなかったこと（曲げない、新素材なし）
5. **名前・位置** — タイトル、アルバム内（後付けなら正直に短く）
6. **余白** — 1文だけ感想（任意、「うれしかった」程度）

## 長さ
400〜900字。スマホで2〜3分。

## DON'T
- 各段落で「なぜ重要か」を説明しない
- 哲学を最後にまとめない

## 例（型のみ）

```text
Suno にプロンプトを入れた。返ってきた2分弱を、そのまま Cubase に読んだ。

ChatGPT でプロンプト文を作り、英訳してから投げる。BPM は揃えることもあるが、出目は毎回違う。

Chopping → Groove Agent → 自分のグリッドに載せ替え。展開を足すときも、別ファイルは探さない。

曲名 inner image は後からついた。イントロの輪郭を考えすぎた週があって、逆に「出たとこ」に寄せた。
```
